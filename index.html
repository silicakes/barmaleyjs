Barmaleyjs

barmaleyjs (pronounced bɑr•mɑ•ləɪ / bar-muh-lay) is a small yet very flexible progressbar plugin written for and with jQuery.

barmaleyjs

Overview

Ever needed a progressbar and the one provided by jQuery-UI just didn't cut it? Meet Barmaley! Barmaley allows you to create a progressbar with several data progressions (referred as data-bars) and access/manage the data through a viewmodel attached to the parent containers $.data() object

Examples

Usage

$("#progressbar").barmaley(options);

Init options

The options parameter is a javascript object that contain one or more of the following:

dataSets - an array of objects that contain the following properties:

  • barLength - [integer/float] The length of the current data-bar in %
  • startFrom - [integer/float] the position in % from which the data-bar will begin
  • barColor - [string] the color of the data-bar
  • Css - [object] an object containing additional CSS properties
  • title - [string] a text that will be placed within the elements title attribute

ticks - [object] an object containing information about creating the ticks in the background (default: false)

  • height - [string] the height of the ticks in px or %
  • amount - [integer] the number of ticks to be displayed
  • responsive - [boolean] defines whther the ticks spacing will be in % or in px. if you want your progress bar to be fluid, set this property to 'true'

Methods

BarmaleyJS has some methods that might help in managing the data. Unless otherwise specified, the syntax will be:

$("#progressbar").barmaley("method",[args | "args"]);

update (id,{options}) - Updates a specific bar using its ID. All the properties of dataBars[{}] are valid

remove (id) - removes the data-bar with the corresponding id

add ({options}) - adds a new bar with a specified set of properties

note that new item ids might vary according to the arrays situation

Accessing data

Accessing data has never been easier since the entire progress bar is represented as a view model stored within the jQuery data() object, under "barData":

$("#progressbar").data("barData")

The above is represented with several helpful attributes:

  • dataBars - [object] an object containing the data regarding the data-bars in addition to an integer id

  • element - the element the method was initiated upon, it might sound redundant, but will be very beneficiary to those who choose to incorporate any kind of MVx framework which uses its own way of manipulating the DOM
